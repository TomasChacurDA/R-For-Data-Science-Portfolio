Modelado - Primeros pasos

Modelar datos sirve principalmente para cuatro cosas:

  1. Cuantificar
  2. Explorar
  3. Resumir
  4. Predecir

Preguntas

  • ¿Cómo cambia el ancho de los pétalos con el largo?
  • ¿Cuál será el ancho del pétalo de una flor con un largo que no hemos visto todavía?

Acá ya no estamos hablando de describir una variable, o la relación entre una variable y una categoría, sino que queremos ver cómo cambia una variable continua con la otra.

¿Qué quiere decir modelar?

  • Hasta acá, solo estamos definiendo relaciones entre variables, pero no estamos diciendo qué tipo de relación es.

  • El factor Otros no es observable. Acá entra la parte probabilítica del modelo, porque vamos a asignarle una determinada distribución (en esta materia, siempre normal)

  • Vammos a hacer una primera suposición:

\[ \text{A} = f(\text{L}, \text{O}) = f(\text{L}) + \text{O} \]

Entra un modelo

Idea: escribamos una expresión matemática que nos parezca que puede ajustarse a la forma en la que se comportan los datos: \[ \text{A} = \mathbf{a} + \mathbf{b}\cdot\text{L} + \text{O} \]

\[ \text{O} \sim N(0, \sigma) \]

¿Cómo lo hicimos?

La familia de funciones definida por el modelo contiene una infinidad de posibles curvas, para cada combinación de los parámetros.

¿Cómo lo hicimos?

Encontramos la mejor reduciendo la dispersión de los residuos al cuadrado. Los residuos son la distancia del modelo a los datos:

\(r = \text{Ancho del pétalo} - \text{modelo}\)

Un toque de matemática

La función que minimizamos para encontrar los valores de \(a\) y \(b\) es:

\[ RMSE = \sqrt{\frac{1}{N}\sum_{i=1}^N {\left\{\text{Petal.Width}_i - \left(\mathbf{a} + \mathbf{b}\cdot \text{Petal.Length}_i\right)\right\}^2}} \] en otras palabras: \[ RMSE = \sqrt{\frac{1}{N}\sum_{i=1}^N {(\text{dato}_i - \text{modelo}_i})^2} = \sqrt{\frac{1}{N}\sum_{i=1}^N r^2_i} \] Minimizando esta función, encontramos los mejores valores de a y b.

Modelo lineal múltiple

Por supuesto, podemos suponer que más de una variable observada afecta al ancho del pétalo.

Si suponemos la misma relación lineal:

\[ \text{A} = \mathbf{a} + \mathbf{b}\cdot\text{L} + \mathbf{c}\cdot x_2 + \text{O}\;\;, \] el modelo es un plano en el espacio de \((L, x_2, A)\).

Modelo lineal múltiple

¿Cómo se hace esto en R?

library(stats)

mod <- lm(formula = Petal.Width ~ Petal.Length,
          data = iris)

¿Cómo se hace esto en R?

mod <- lm(formula = Petal.Width ~ Petal.Length,

¿Cómo se hace esto en R?

library(stats)

mod <- lm(formula = Petal.Width ~ Petal.Length,
          data = iris)

Acá, se usa una mini-sitaxis propia de la función lm:

  • La virgulilla ~ representa una relación lineal entre las variables. En este caso, \(Petal.Width = \mathbf{a} + \mathbf{b}\times Petal.Length\)
  • La suma + agrega variables independientes al modelo. Por ejemplo,
formula = Petal.Width ~ Petal.Length + Sepal.Length

\(Petal.Width = \mathbf{a} + \mathbf{b}\times Petal.Length + \mathbf{c} \times Sepal.Length\)

  • En todos lo casos, aparece el término independiente (sesgo u ordenada al origen), \(\mathbf{a}\). Para quitarla, hay que hacerlo explícitamente
formula = Petal.Width ~ Petal.Length - 1
  • El producto * representa interacciones entre las variables (tema para dentro de dos semanas)
