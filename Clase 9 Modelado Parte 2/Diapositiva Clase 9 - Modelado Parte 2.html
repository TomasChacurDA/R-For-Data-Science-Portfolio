Términos de interacción

Una forma de flexibilizar el modelo es incluyendo términos de interacción, para generar un modelo diferente para cada clase.

Esto se logra con la inclusión de * en la formula del modelo.

P.ej.:

  • hwy_kml ~ poly(displ, 2, raw=T) * class)

  • Para incluir cambios solo en el término lineal del modelo (además del offset)
    hwy_kml ~ displ * class + I(displ**2))

  • Para incluir cambios solo en el término cuadrático del modelo (además del offset)
    hwy_kml ~ displ + I(displ**2) * class)

  • Para no permitir cambios en el offset
    hwy_kml ~ displ + I(displ**2) * class - class)

Comparación de modelos

En general, es de esperar que un modelo mejore a medida que incluímos más parámetros.

Sin embargo, queremos tener el modelo más simple posible que funcione bien.

  • Mayor explicabilidad.
  • Comunicación más sencilla.

Queremos una forma de cuantificar cuándo agregar más parámetros “deja de valer la pena”.

Idea 1. R2 ajustado. Poco contenido probabilístico

Idea 2. Test F.

Comparación de modelos. Test F

La idea del test F es comparar los residuos de dos modelos: uno sencillo con \(p_1\) parámetros y otro con \(p_2 > p_1\) parámetros, y con expresiones (fórmulas) \(f_1\) y \(f_2\).

Suponiendo que llamamos \(y\) a la variable target y \(x\) son las covariables, se calcula (para el modelo 1, por ejemplo):

\[RSS_1 = (y_1 - f_1(x_1))^2 + (y_2 - f_1(x_2))^2 + \ldots + (y_N - f_1(x_N))^2\] \[RSS_1 = \sum_{i=1}^N (y_i - f_1(x_i))^2\]

Entonces, el estadístico para el test es

\[ F = \frac{\left(\frac{RSS_1 - RSS_2}{p_2 - p_1}\right)}{\frac{RSS_2}{N - p_2}}\;\;.\]

Comparación de modelos. Test F

\[ F = \frac{\left(\frac{RSS_1 - RSS_2}{p_2 - p_1}\right)}{\frac{RSS_2}{N - p_2}}\;\;.\]

En el caso en que ambos modelos son igualmente buenos para explicar los datos, podemos calcular la probabilidad de obtener un determinado valor de \(F\) igual o mayor al observado en los datos \(F_o\):

\[p(F > F_o)\] Si esta probabilidad (\(p\)-valor) es muy chiquito, podemos descartar la hipótesis de que ambos modelos son iguales. La práctica en general es entonces preferir el modelo más complejo.

En R:

mod1 <- lm(data=df, hwy_kml ~ poly(displ,2, raw=TRUE) + class)
mod2 <- lm(data=df, hwy_kml ~ poly(displ,2, raw=TRUE) * class)

anova(mod1, mod2)

Variables confundidoras

Pero hay otra variable que tiene una fuerte relación con el price; son los quilates (carat).

ggplot(diamonds) + geom_point(aes(x=log(carat), y=log(price)))

Variables confundidoras

Si miramos los residuos de un ajuste con carat, encontramos lo que uno podría esperar

Moraleja

  • Este es un ejemplo de la paradoja de Simpson: el efecto de una variable sobre el target está modificado de tal manera por otra variable que el efecto parece el contrario.

  • Por lo tanto, para ver el efecto en el color, sin la confusión que aporta la variable de los quilates, tenemos que “controlar” por esta variable carat;

formula = log(price) ~ color + log(carat)

  • Es decir, usar términos adicionales en el modelo no solo tiene el interés de obtener un mejor ajuste, sino que también nos permite encontrar correctamente el efecto de una variable sobre la variable target.

  • Claro que podrán preguntar: ¿No podría haber otras variables confundidoras que no identificamos?
